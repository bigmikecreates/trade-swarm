## Goal
Take the working backtest from v0.1.0 and run it live against real market data using a paper trading account. No real money. The system must place, monitor, and log orders automatically without you touching it.

- Backtest Sharpe ratio > 0.8 on 2 years of data ✓
- Max drawdown < 20% in backtest ✓
- Single `TrendSignalAgent` working reliably ✓

## Gate to exit this version
- 4 continuous weeks of paper trading with no critical bugs
- All orders logged correctly to SQLite
- P&L dashboard showing accurate numbers
- Kill switch tested and confirmed working

### 2. Broker adapter

```python
# brokers/alpaca_adapter.py
import alpaca_trade_api as tradeapi
from dataclasses import dataclass
from datetime import datetime

@dataclass
class OrderResult:
    order_id:    str
    asset:       str
    direction:   str
    size:        float
    status:      str
    filled_price: float | None
    timestamp:   datetime

class AlpacaAdapter:
    def __init__(self, api_key: str, secret: str, base_url: str):
        self.api = tradeapi.REST(api_key, secret, base_url)

    def place_market_order(self, asset: str, qty: float, side: str) -> OrderResult:
        order = self.api.submit_order(
            symbol        = asset,
            qty           = qty,
            side          = side,           # "buy" or "sell"
            type          = "market",
            time_in_force = "gtc",
        )
        return OrderResult(
            order_id     = order.id,
            asset        = asset,
            direction    = side,
            size         = qty,
            status       = order.status,
            filled_price = None,            # polled separately
            timestamp    = datetime.utcnow()
        )

    def get_order_status(self, order_id: str) -> dict:
        o = self.api.get_order(order_id)
        return {
            "status":     o.status,
            "filled_qty": float(o.filled_qty or 0),
            "avg_price":  float(o.filled_avg_price or 0),
        }

    def cancel_order(self, order_id: str):
        self.api.cancel_order(order_id)

    def get_positions(self) -> list[dict]:
        return [
            {"asset": p.symbol, "qty": float(p.qty),
             "avg_price": float(p.avg_entry_price)}
            for p in self.api.list_positions()
        ]

    def get_account_equity(self) -> float:
        return float(self.api.get_account().equity)
```

### 3. Position sizing

Never risk more than 1% of account on a single trade. This must be wired before any order is placed.

```python
# risk/position_sizer.py

def calculate_position_size(
    account_equity:  float,
    entry_price:     float,
    stop_loss_price: float,
    risk_pct:        float = 0.01    # 1% of account
) -> float:
    """
    Returns number of shares/units to buy.
    Max loss = (entry - stop) × units = account × risk_pct
    """
    max_loss     = account_equity * risk_pct
    risk_per_unit = abs(entry_price - stop_loss_price)

    if risk_per_unit == 0:
        return 0.0

    return round(max_loss / risk_per_unit, 4)
```

### 4. Basic risk gate (4 rules only)

```python
# risk/gate.py
import redis, json
from dataclasses import dataclass
from datetime import datetime
from enum import Enum

class GateResult(Enum):
    PASS  = "pass"
    BLOCK = "block"
    HALT  = "halt"

@dataclass
class GateDecision:
    result:     GateResult
    rule:       str | None
    reason:     str | None
    timestamp:  datetime

class BasicRiskGate:
    """
    v0.2.0 gate — 4 rules only.
    Full gate implemented in v0.5.0.
    """
    def __init__(self, account_equity: float):
        self.equity       = account_equity
        self.daily_start  = account_equity
        self.r            = redis.Redis(decode_responses=True)

    def check(self, order: dict, current_equity: float) -> GateDecision:
        # Rule 1: kill switch
        if self.r.get("kill_switch") == "1":
            return GateDecision(GateResult.HALT, "kill_switch",
                "Kill switch active", datetime.utcnow())

        # Rule 2: daily loss limit (3%)
        daily_loss = (self.daily_start - current_equity) / self.daily_start
        if daily_loss >= 0.03:
            return GateDecision(GateResult.HALT, "daily_loss",
                f"Down {daily_loss:.1%} today", datetime.utcnow())

        # Rule 3: max position size (10% of account)
        notional  = order.get("size", 0) * order.get("price", 0)
        pos_pct   = notional / current_equity if current_equity > 0 else 1.0
        if pos_pct > 0.10:
            return GateDecision(GateResult.BLOCK, "max_position",
                f"Order is {pos_pct:.1%} of account", datetime.utcnow())

        # Rule 4: duplicate order check
        asset        = order.get("asset")
        existing_dir = self.r.get(f"position_direction:{asset}")
        if existing_dir and existing_dir == order.get("direction") \
                and order.get("action") == "enter":
            return GateDecision(GateResult.BLOCK, "duplicate",
                f"Already {existing_dir} {asset}", datetime.utcnow())

        return GateDecision(GateResult.PASS, None, None, datetime.utcnow())

    def activate_kill_switch(self):
        self.r.set("kill_switch", "1")
        print("⛔ Kill switch activated")

    def reset_daily(self, current_equity: float):
        """Call this at market open each day."""
        self.daily_start = current_equity
```

### 5. Trade logger (SQLite)

```python
# logging/trade_log.py
import sqlite3, json
from datetime import datetime
from pathlib import Path

DB_PATH = Path("data/trades.db")

def init_db():
    DB_PATH.parent.mkdir(exist_ok=True)
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        CREATE TABLE IF NOT EXISTS trades (
            id            INTEGER PRIMARY KEY AUTOINCREMENT,
            timestamp     TEXT,
            asset         TEXT,
            direction     TEXT,
            action        TEXT,
            size          REAL,
            entry_price   REAL,
            exit_price    REAL,
            pnl           REAL,
            stop_loss     REAL,
            order_id      TEXT,
            status        TEXT,
            indicators    TEXT    -- JSON blob for audit
        )
    """)
    conn.commit()
    conn.close()

def log_trade(trade: dict):
    conn = sqlite3.connect(DB_PATH)
    conn.execute("""
        INSERT INTO trades
        (timestamp, asset, direction, action, size, entry_price,
         exit_price, pnl, stop_loss, order_id, status, indicators)
        VALUES (?,?,?,?,?,?,?,?,?,?,?,?)
    """, (
        datetime.utcnow().isoformat(),
        trade.get("asset"),
        trade.get("direction"),
        trade.get("action"),
        trade.get("size"),
        trade.get("entry_price"),
        trade.get("exit_price"),
        trade.get("pnl"),
        trade.get("stop_loss"),
        trade.get("order_id"),
        trade.get("status"),
        json.dumps(trade.get("indicators", {})),
    ))
    conn.commit()
    conn.close()
```

### 6. The main trading loop

```python
# main.py
import time
from datetime import datetime
from data.fetcher import fetch_ohlcv
from agents.signal.trend_agent import TrendSignalAgent, Direction
from brokers.alpaca_adapter import AlpacaAdapter
from risk.gate import BasicRiskGate
from risk.position_sizer import calculate_position_size
from logging.trade_log import init_db, log_trade
from config import ALPACA_API_KEY, ALPACA_SECRET_KEY, ALPACA_BASE_URL

SYMBOL        = "AAPL"
CHECK_INTERVAL = 300   # check every 5 minutes

def run():
    init_db()
    broker  = AlpacaAdapter(ALPACA_API_KEY, ALPACA_SECRET_KEY, ALPACA_BASE_URL)
    agent   = TrendSignalAgent(SYMBOL)
    equity  = broker.get_account_equity()
    gate    = BasicRiskGate(account_equity=equity)

    print(f"Starting paper trading loop. Equity: ${equity:,.2f}")

    while True:
        try:
            # 1. Fetch latest data
            df     = fetch_ohlcv(SYMBOL, period="60d", interval="1h")
            signal = agent.generate(df)
            equity = broker.get_account_equity()

            print(f"{datetime.utcnow()} | {signal.direction.value} | "
                  f"strength={signal.strength:.2f} confidence={signal.confidence:.2f}")

            # 2. Only act on strong signals
            if signal.strength * signal.confidence < 0.4:
                print("Signal too weak — skipping")
                time.sleep(CHECK_INTERVAL)
                continue

            # 3. Check current positions
            positions = broker.get_positions()
            in_trade  = any(p["asset"] == SYMBOL for p in positions)

            # 4. Entry logic
            if not in_trade and signal.direction == Direction.LONG:
                price     = float(df["Close"].iloc[-1])
                atr       = float(df["ATRr_14"].iloc[-1]) if "ATRr_14" in df.columns else price * 0.01
                stop_loss = price - (2 * atr)
                size      = calculate_position_size(equity, price, stop_loss, risk_pct=0.01)

                order = {"asset": SYMBOL, "direction": "buy",
                         "action": "enter", "size": size, "price": price}
                decision = gate.check(order, equity)

                if decision.result.value == "pass":
                    result = broker.place_market_order(SYMBOL, size, "buy")
                    log_trade({**order, "entry_price": price, "stop_loss": stop_loss,
                               "order_id": result.order_id, "status": "open",
                               "indicators": signal.indicators})
                    print(f"✅ Opened LONG {size} {SYMBOL} @ ${price:.2f}")
                else:
                    print(f"🚫 Gate blocked: {decision.reason}")

            # 5. Exit logic — flat signal or direction reversal
            elif in_trade and signal.direction in (Direction.FLAT, Direction.SHORT):
                result = broker.place_market_order(SYMBOL, positions[0]["qty"], "sell")
                log_trade({"asset": SYMBOL, "direction": "sell", "action": "exit",
                           "size": positions[0]["qty"], "order_id": result.order_id,
                           "status": "closed", "indicators": signal.indicators})
                print(f"✅ Closed position in {SYMBOL}")

        except Exception as e:
            print(f"⚠️  Error: {e}")

        time.sleep(CHECK_INTERVAL)

if __name__ == "__main__":
    run()
```

# dashboard/app.py
import streamlit as st
import sqlite3, pandas as pd
from pathlib import Path

st.set_page_config(page_title="Trading System", layout="wide")
st.title("Paper Trading Dashboard")

conn  = sqlite3.connect("data/trades.db")
df    = pd.read_sql("SELECT * FROM trades ORDER BY timestamp DESC", conn)
conn.close()

if df.empty:
    st.info("No trades yet.")
else:
    total_pnl    = df["pnl"].sum()
    win_rate     = (df["pnl"] > 0).mean() * 100
    total_trades = len(df)

    col1, col2, col3 = st.columns(3)
    col1.metric("Total P&L",    f"${total_pnl:,.2f}")
    col2.metric("Win Rate",     f"{win_rate:.1f}%")
    col3.metric("Total Trades", total_trades)

    st.subheader("Trade History")
    st.dataframe(df[["timestamp","asset","direction","size",
                      "entry_price","exit_price","pnl","status"]])

    st.subheader("Cumulative P&L")
    df["cumulative_pnl"] = df["pnl"].cumsum()
    st.line_chart(df.set_index("timestamp")["cumulative_pnl"])

## Infrastructure: Docker Compose for Redis

Redis runs via Docker Compose — no local install or WSL required. Simplifies setup on Windows and matches the eventual v0.5.0 Docker stack.

```yaml
# docker-compose.yml
services:
  redis:
    image: redis:alpine
    ports:
      - "6379:6379"
```

```bash
docker compose up -d    # Start Redis in background
docker compose down     # Stop when done
```

The paper trading loop connects to `redis://localhost:6379` (or `REDIS_URL` in `.env`). Same config works locally and on a VPS.

## Deployment workflow

**Phase 1 — Local validation (recommended: ~1 day)**  
Run the full stack on your machine to confirm everything works before committing to a 4-week run:

1. `docker compose up -d` (Redis)
2. Copy `.env.example` to `.env`, add Alpaca keys
3. `pip install -e .` and `trade-swarm-paper`
4. Let it run for at least one full trading day
5. Verify: orders logged, dashboard accurate, kill switch works

**Phase 2 — VPS for 4-week gate**  
The gate requires 4 continuous weeks. A laptop is unreliable (sleep, power, updates). Use a cloud VPS:

1. Provision a small instance (DigitalOcean, Linode, Hetzner — ~$5–10/mo)
2. SSH in, clone repo, `pip install -e .`
3. Run Redis via Docker Compose (or `apt install redis-server`)
4. Run the loop in `tmux` or `screen`, or as a systemd service
5. Leave it running for 4 weeks

## VPS latency

Running the paper loop on a VPS adds latency between your process and Alpaca’s API. For this strategy it is negligible:

- **Strategy:** Daily EMA crossover on SPY. Signals are based on end-of-day bars. No intraday scalping.
- **Check interval:** 5 minutes. A few hundred ms of extra round-trip to Alpaca does not affect which bar we see or when we decide.
- **Order execution:** Market orders. Fill price is determined by the exchange, not by our submission speed. A 50–200 ms delay from a US-based VPS to Alpaca is irrelevant for daily signals.
- **When it would matter:** High-frequency, tick-level, or sub-second strategies. v0.2.0 is not that.

**Recommendation:** Use a VPS in the same region as Alpaca (e.g. US East) if available. Latency will be low and inconsequential for daily paper trading.

## Advanced position management (deferred)

**v0.2.0 scope:** One open position per symbol; size from risk-to-stop + notional cap; **full close** when the signal flips (flat / opposite). That is enough to validate Alpaca paper, gates, logging, and kill switch without scope creep.

**What “advanced” means here (examples):** adding to winners, partial take-profits, laddered exits, or rules that depend on **aggregate** exposure across legs — not implemented in v0.2.0.

**Where it fits in the roadmap:** Implement in a **subsequent version after v0.2.0**, not as a silent expansion of this gate. The planned home is **v0.4.0** (orchestrator + full `RiskAgent` + **2–3 assets** in `v0.4.0/v0.4.0.md`) — the layer where portfolio-level sizing and conflict resolution already live. **v0.3.0** adds a second agent and regime on **one asset still**; multi-asset expansion is specified for **v0.4.0**, not v0.3.0.
